Temas 1.1 a 1.1.c

Presentado por: @clouddataproc

1.1 Solución Estratificada de Problemas

Consiste en dividir la infraestructura TIC en capas (estratos) para aislar fallos:

  • Hardware: Recursos físicos.
  • Abstracción: Capa de virtualización.
  • Sistema: SO y librerías.
  • Aplicación: Software final.

Beneficio: Localización precisa de errores y gestión modular.

1.1.a Interpretación Pura

Descripción

Traduce cada instrucción del sistema invitado al host, una por una.

Características

  • Compatibilidad total de arquitectura.
  • Rendimiento lento (alto overhead).

Casos y Ejemplos

  • Emulación de hardware antiguo.
  • Debugging de kernels.
  • Software: Bochs, QEMU (modo TCG).

1.1.b Recompilación Dinámica

Descripción

Traduce bloques de código binario en tiempo real y los guarda en caché.

Características

  • Mucho más rápido que la interpretación.
  • Traducción Just-In-Time (JIT).

Casos y Ejemplos

  • Máquinas virtuales de lenguajes.
  • Software legacy en arquitecturas nuevas.
  • Software: JVM (Java), Rosetta (Apple).

1.1.c Hipervisión (Bare Metal)

Descripción

El hipervisor se instala directamente sobre el hardware físico.

Características

  • Máximo rendimiento y eficiencia.
  • Seguridad mejorada (sin SO anfitrión).

Casos y Ejemplos

  • Datacenters masivos.
  • Nubes públicas (IaaS).
  • Software: VMware ESXi, Proxmox, Hyper-V.
